Resume Parser

Intelligent Filtering of Resumes Using Feature-Based Analysis

"We hypothesize that combining AI-powered parsing, NLP-based feature extraction, and customizable filters can automatically transform unstructured resumes into structured profiles and enable accurate, bias-reduced candidate shortlisting for recruiters."

Screening Support Tool – Final Hiring Decision Remains with Recruiters

Problem Statement

Recruiters often face thousands of applications per job opening, making manual resume screening time-consuming, inconsistent, and prone to bias. Heterogeneous resume formats and ambiguous data points make it difficult to reliably identify the best candidates.

  • Format Heterogeneity: PDFs, DOCX, scanned images, and graphical layouts confuse basic parsers
  • Inconsistency & Bias: Manual screening is tedious and influenced by unconscious bias
  • Data Ambiguity: Overlapping employment dates, missing information, and synonym skills
  • Scaling Issues: High volume of resumes slows down the hiring pipeline

→ Need for an automated, accurate, and feature-based resume parsing and filtering system

Literature Review / Market Research

Existing Solutions:

  • Traditional keyword-based parsers (limited context, high false matches)
  • ATS tools focusing on simple text matching and format-dependent parsing
  • Third-party resume parsing APIs with limited transparency and customization
  • AI/NLP based tools that still lack fine-grained feature-based filtering

Industry trend: AI parsing can reduce time-to-hire by up to 75% and improve matching accuracy by around 40% when properly integrated into workflows.

Research Gap / Innovation

Our Unique Approach:

  • ✓ Feature-Based Filtering: Multi-criteria filtering on skills, experience, education, and certifications
  • ✓ Deep Data Extraction: Extraction of 150+ structured fields from diverse resume formats
  • ✓ Context-Aware NLP: Handles synonyms and role-specific vocabulary
  • ✓ Format Standardization: Converts all resumes into a unified JSON/XML schema
  • ✓ Inconsistency Flagging: Detects employment gaps and overlapping job dates

Dataset / Input

  • Sources: Resumes in PDF, DOCX, TXT, and image formats
  • Preprocessing: OCR for scanned/image resumes, text cleaning
  • Fields: Name, contact, education, skills, experience, projects, certifications
  • Normalization: Standard date formats, role titles, and skill naming
  • Storage: Structured JSON objects per candidate

Model / Architecture

  • NLP Stack: Tokenization, POS tagging, and Named Entity Recognition
  • Skill Taxonomy: Mapping synonyms and related skills into unified groups
  • Rule + ML Hybrid: Rules for mandatory fields and ML-based scoring for ranking
  • Scoring Engine: Weighted feature-based match score for each candidate
  • Error Handling: Detection of missing, ambiguous, or conflicting data

Tech Stack

  • Python (core parsing engine)
  • spaCy / NLTK for NLP
  • Tesseract / OpenCV for OCR
  • Flask / FastAPI REST API
  • MongoDB / PostgreSQL for structured storage
VIEW CODE / DEMO

Live Execution Flow

  1. 1. Recruiter Onboarding: Login, job role selection, and feature preference setup.
  2. 2. Resume Upload: Bulk upload of resumes in multiple formats.
  3. 3. Parsing & Extraction: Engine performs OCR (if needed), NLP, and feature extraction into structured fields.
  4. 4. Feature-Based Filtering: Filters applied on skills, years of experience, education, and certifications.
  5. 5. Ranked Output: Candidates are ranked by match score with flags for gaps or overlaps.
  6. 6. Export / Integration: Shortlisted profiles exported or integrated with ATS/HR systems.
